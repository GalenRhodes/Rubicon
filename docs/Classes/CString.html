CString

open class CString

This is a class to help with the conversion between C strings and the Swift String class. One note about the use of signed and unsigned chars. Some C functions make a distinction between a signed char (usually the default - when only the lower 7-bits are significant) and an unsigned char (usually in conjunction with the UTF-8 character encoding scheme). This class has methods to handle both.

  • newUtf8BufferOf(length:_:)

    Create a new CString and do something with it right away. So, for example, instead of having to do this:

       let buffer = UnsafeMutablePointer.allocate(capacity: 256)
   defer { buffer.deallocate() }
   guard strerror_r(code, buffer, 255) == 0 else { return "Unknown Error: \(code)" }
   let str = String(utf8String: buffer) ?? "Unknown Error: \(code)"

    You can, instead, do this:

        let str = (CString.newUtf8BufferOf(length: 255) { ((strerror_r(code, $0, $1) == 0) ? strlen($0) : -1) })?.string ?? "Unknown Error: \(code)"

    Declaration

    Swift

    public static func newUtf8BufferOf(length: Int, _ block: (BytePointer, Int) throws -> Int) -> CString?

    Parameters

    length

    The length of the buffer to create NOT INCLUDING the nil terminator.
    block

    The closure that will get executed. The buffer and it’s length, NOT INCLUDING the nil terminator, will be passed to the closure. Upon success the closure should return the actual length of the C string in the buffer, NOT INCLUDING, the nil terminator. If there was an error then the closure should return -1 to indicate that the buffer does not contain a valid C string.

    Return Value

    An instance of CString or nil if -1 was returned from the closure.
